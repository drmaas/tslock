# Spec: @tslock/s3

## Overview

The `@tslock/s3` package provides a distributed lock provider backed by Amazon S3 (and S3-compatible object stores such as LocalStack, MinIO, R2, etc.). It implements the `StorageAccessor` interface and wraps it with the shared `StorageBasedLockProvider` from `@tslock/core`.

Each lock is represented as a single S3 object whose **user metadata** carries the lock fields (`lockUntil`, `lockedAt`, `lockedBy`). Mutual exclusion is achieved through S3 **conditional writes**: `PutObject` with `IfNoneMatch: "*"` for inserts (create-if-not-exists) and `IfMatch: <ETag>` for updates (compare-and-swap on the current ETag). An S3 `HeadObject` precedes each conditional put to fetch the current ETag and metadata.

This is a direct port of ShedLock's `S3StorageAccessor` / `S3LockProvider` to TypeScript using the AWS SDK v3 (`@aws-sdk/client-s3`).

## Public API

### 1. S3LockProvider

```typescript
class S3LockProvider implements ExtensibleLockProvider {
  constructor(s3: S3Client, config: S3ProviderConfig);
  lock(config: LockConfiguration): Promise<SimpleLock | undefined>;
  clearCache(name: string): void;
}
```

Constructs an `S3StorageAccessor` with the given client and config, then wraps it in a `StorageBasedLockProvider`. All locking semantics (insert-then-update, `LockRecordRegistry`, `StorageLock` unlock/extend) come from `StorageBasedLockProvider` — see `@tslock/core` spec §13. The S3 provider only supplies the four `StorageAccessor` operations.

### 2. S3ProviderConfig

```typescript
interface S3ProviderConfig {
  /** S3 bucket name. Required. */
  bucket: string;
  /** Key prefix applied to every lock object. Default: "shedlock/". */
  objectPrefix?: string;
  /**
   * Custom endpoint URL (e.g. "http://localhost:4566" for LocalStack,
   * S3-compatible stores like MinIO/R2). Optional. Forwarded to the SDK's
   * S3Client via its `endpoint` config, not stored here — but documented for
   * the user's convenience when constructing the client.
   */
  endpoint?: string;
}
```

**Defaults & validation:**
- `objectPrefix` defaults to `"shedlock/"`.
- `bucket` must be a non-empty string. Throws `LockException` if missing.
- `endpoint` is informational only — the user constructs their own `S3Client` (so they control region, credentials, forcePathStyle, endpoint, etc.). The provider reads `bucket` and `objectPrefix` from `S3ProviderConfig`.

**Object key layout:**

```
<objectPrefix><lockName>
e.g. "shedlock/my-task-name"
```

Lock names are used verbatim as the suffix. Callers should ensure lock names are valid S3 key components (no leading `/`, reasonable length).

### 3. S3LockProviderConfig (factory helper)

```typescript
function createS3ProviderConfig(input: {
  bucket: string;
  objectPrefix?: string;
  endpoint?: string;
}): S3ProviderConfig;
```

Validates and applies defaults. Equivalent to passing a literal object, but provides a single typed entry point with validation.

## Locking Mechanism

The lock record for a given `name` is a single S3 object:

| Field | S3 metadata key | Value |
|---|---|---|
| `lockUntil` | `lockUntil` | ISO-8601 instant string (3-digit millis), e.g. `2018-12-07T12:30:37.810Z` |
| `lockedAt` | `lockedAt` | ISO-8601 instant string |
| `lockedBy` | `lockedBy` | Hostname string (`Utils.getHostname()`) |

The object body is empty (`""`). All state lives in user metadata. The S3 **ETag** serves as the optimistic-concurrency token: a successful `PutObject` with `IfMatch: <ETag>` only succeeds when the current object's ETag matches, implementing compare-and-swap.

### insertRecord(config)

Invoked by `StorageBasedLockProvider.lock()` when the `LockRecordRegistry` has no record for `name` (first-ever attempt at this name, or after a cache clear).

1. `HeadObjectCommand({ Bucket, Key })` — check whether the object already exists.
2. If the response succeeds (object exists) → return `false`. The registry will be marked as created, and `lock()` will fall through to `updateRecord`.
3. If `HeadObject` throws with name `NotFound` (HTTP 404) → the object does not exist.
   - (Any other error — connection, 403, etc. — propagates as a storage error.)
4. `PutObjectCommand({
     Bucket,
     Key,
     Body: "",
     Metadata: {
       lockUntil: Utils.toIsoString(lockAtMostUntil(config)),
       lockedAt:  Utils.toIsoString(config.createdAt),
       lockedBy:  this.getHostname(),
     },
     IfNoneMatch: "*",
   })`
5. If `PutObject` succeeds → return `true`.
6. If `PutObject` throws `PreconditionFailed` (HTTP 412) → someone else created the object concurrently. Return `false`.
   - (Any other error propagates.)

**`IfNoneMatch: "*"`** is S3's create-if-not-exists conditional. It is supported on the standard S3 endpoint and on most S3-compatible stores (LocalStack ≥ 2.x, MinIO, R2). If the target store does not support it, `PutObject` will 412 or 400 on every call — caller must configure a compatible endpoint.

### updateRecord(config)

Invoked when `LockRecordRegistry` knows a record exists for `name`.

1. `HeadObjectCommand({ Bucket, Key })` → fetch current `ETag` and `Metadata`.
2. If `NotFound` (404):
   - Return `false`. (Should not happen if the registry is consistent, but handle it — the object may have been deleted externally between the registry being populated and now. The caller — `StorageBasedLockProvider.lock()` — will clear the cache on a subsequent exception path; for a clean `false` return, the lock simply appears held/missing.)
3. Parse `lockUntil` from `Metadata.lockUntil`.
   - If missing or unparseable → propagate `LockException` (corrupted record).
4. If `lockUntil > ClockProvider.now()` → return `false` (still locked by someone else).
5. `PutObjectCommand({
     Bucket,
     Key,
     Body: "",
     Metadata: {
       lockUntil: Utils.toIsoString(lockAtMostUntil(config)),
       lockedAt:  Utils.toIsoString(config.createdAt),
       lockedBy:  this.getHostname(),
     },
     IfMatch: <ETag from step 1>,
   })`
6. If `PutObject` succeeds → return `true`.
7. If `PutObject` throws `PreconditionFailed` (HTTP 412) → the object was modified concurrently. Return `false`.

### unlock(config)

Invoked by `StorageLock.doUnlock()`. Sets `lockUntil` to the unlock time so another caller can immediately `updateRecord` past it.

1. `HeadObjectCommand({ Bucket, Key })` → fetch `ETag` (and current metadata so `lockedAt` / `lockedBy` are preserved).
2. If `NotFound` (404) → no-op return (record was already gone — log a warning, do not throw; the lock is already effectively released).
3. `PutObjectCommand({
     Bucket,
     Key,
     Body: "",
     Metadata: {
       lockUntil: Utils.toIsoString(unlockTime(config)),
       lockedAt:  Metadata.lockedAt ?? Utils.toIsoString(config.createdAt),
       lockedBy:  Metadata.lockedBy ?? this.getHostname(),
     },
     IfMatch: <ETag>,
   })`
4. If `PreconditionFailed` (412) → someone else modified the object concurrently. The lock record is in an unknown state. Best-effort: log warning, do not throw (the lock will expire via the original `lockAtMostFor`).

**`unlockTime(config)`** is `Math.max(ClockProvider.now(), lockAtLeastUntil(config))` — honors `lockAtLeastFor`.

### extend(config)

Invoked by `StorageLock.doExtend()`. Re-extends the lock's `lockUntil` to `lockAtMostUntil(newConfig)` **only if** the caller is still the current holder.

1. `HeadObjectCommand({ Bucket, Key })` → fetch `ETag` + `Metadata`.
2. If `NotFound` (404) → return `false` (record gone — lock was deleted).
3. Parse `lockUntil` and `lockedBy` from metadata.
4. If `lockedBy !== this.getHostname()` → return `false` (lock held by someone else; do not extend).
5. If `lockUntil <= ClockProvider.now()` → return `false` (lock already expired; cannot extend).
6. `PutObjectCommand({
     Bucket,
     Key,
     Body: "",
     Metadata: {
       lockUntil: Utils.toIsoString(lockAtMostUntil(config)),
       lockedAt:  Metadata.lockedAt,
       lockedBy:  Metadata.lockedBy,
     },
     IfMatch: <ETag>,
   })`
7. If `PutObject` succeeds → return `true`.
8. If `PreconditionFailed` (412) → return `false` (concurrent modification).

`lockedAt` is preserved from the existing metadata (the original acquisition time, not the extension time). `lockedBy` is preserved.

## Error Handling

| Situation | Behavior |
|---|---|
| `HeadObject` returns `NotFound` (404) in `insertRecord` | Proceed to `PutObject` (object does not exist — proceed with insert) |
| `HeadObject` returns `NotFound` (404) in `updateRecord` | Return `false` |
| `HeadObject` returns `NotFound` (404) in `unlock` | Log warning, no-op return |
| `HeadObject` returns `NotFound` (404) in `extend` | Return `false` |
| `PutObject` with `IfNoneMatch` returns `PreconditionFailed` (412) | Return `false` (object already exists) |
| `PutObject` with `IfMatch` returns `PreconditionFailed` (412) | Return `false` (ETag mismatch — concurrent modification) |
| `PutObject` returns any other SDK error (`S3ServiceException` not 412/404) | Propagate as storage error |
| `HeadObject` returns any non-404 error | Propagate as storage error |
| Corrupt metadata (`lockUntil` unparseable) | Propagate `LockException('Corrupted lock record: ...')` |
| Missing `bucket` in config | Throw `LockException` at provider construction |
| AWS credentials missing / STS expired | Propagate SDK `CredentialsProviderError` / `AccessDenied` |

**Error detection strategy:** AWS SDK v3 throws `S3ServiceException` subclasses with a `$metadata.httpStatusCode` field and a `name` property. The accessor maps:
- `name === 'NotFound'` or `$metadata.httpStatusCode === 404` → not-found branch.
- `name === 'PreconditionFailed'` or `$metadata.httpStatusCode === 412` → conditional-fail branch.
- All others propagate.

Do **not** rely on the HTTP status code alone — some S3-compatible backends return `name` values that differ. Match on either `name` or status code.

## Integration Test Approach

LocalStack provides S3 emulation sufficient for TSLock's needs (HeadObject, PutObject with `IfNoneMatch` / `IfMatch`, ETag generation, user metadata). Two options:

1. **`testcontainers`** (preferred): spin up `localstack/localstack:s3-latest` in Docker. Read the endpoint URL from the container's host port. Construct `new S3Client({ endpoint, region: 'us-east-1', credentials: { accessKeyId: 'test', secretAccessKey: 'test' }, forcePathStyle: true })`.
2. **`docker-compose.localstack.yml`**: a static compose file the developer can `docker compose up -d` before running tests. Useful for IDE-integrated test runs.

The integration test extends `storageBasedLockProviderIntegrationTests` from `@tslock/test-support`. It uses **real time advancement** (`sleep()` from test-support) because LocalStack honors real wall-clock time for ETag and metadata sequencing.

**`forcePathStyle: true`** is mandatory for LocalStack and most S3-compatible stores — virtual-hosted-style URLs require a DNS entry that resolves to the emulator.

**Caveats:**
- LocalStack's ETag is a stable hash of object content + metadata. Since our body is empty, the ETag changes only with metadata — which is exactly what we want for compare-and-swap.
- Real S3 may eventually provide strong consistency for conditional writes; LocalStack emulates this. If a future LocalStack version regresses on conditional-put semantics, the integration tests will catch it (412s will fire at the wrong time).

## Non-Goals (for this package)

- No bucket creation or lifecycle management — the bucket must pre-exist.
- No S3 Object Lock / Glacier / versioning integration — plain PUT/HEAD only.
- No multi-region replication handling.
- No KMS encryption configuration — the user's `S3Client` configuration governs encryption.
- No automatic cleanup of stale lock objects. Lock objects persist indefinitely; the metadata `lockUntil` simply reads as expired. Users wanting cleanup should set an S3 Lifecycle rule on `objectPrefix` (e.g. expire objects under `shedlock/` after N days).
