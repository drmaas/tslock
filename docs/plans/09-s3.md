# Implementation Plan: @tslock/s3

## Overview

Build the `@tslock/s3` package — a distributed lock provider backed by Amazon S3 (and S3-compatible stores). It implements `StorageAccessor` over `@aws-sdk/client-s3` and wraps it with `StorageBasedLockProvider` from `@tslock/core`. The locking primitive is S3 conditional writes: `IfNoneMatch: "*"` for create-if-not-exists and `IfMatch: <ETag>` for compare-and-swap, both preceded by `HeadObject` to fetch the current ETag and metadata.

This is a greenfield implementation. No code exists yet.

## Prerequisites

- `@tslock/core` built and available in the pnpm workspace (provides `StorageBasedLockProvider`, `StorageAccessor`, `AbstractStorageAccessor`, `AbstractSimpleLock`, `LockException`, `Utils`, `ClockProvider`, `LockConfiguration`, `lockAtMostUntil`, `unlockTime`).
- `@tslock/test-support` built and available (provides `storageBasedLockProviderIntegrationTests`, `config`, `sleep`).
- Docker available locally for LocalStack integration tests (or `testcontainers` can pull the image).
- `pnpm-workspace.yaml` includes `packages/*`.

### Step 3: Implement S3 error helpers

**File:** `src/s3-errors.ts`

```typescript
import { S3ServiceException } from '@aws-sdk/client-s3';

function isNotFound(e: unknown): boolean;
function isPreconditionFailed(e: unknown): boolean;
```

- `isNotFound`: `e instanceof S3ServiceException && (e.name === 'NotFound' || e.$metadata?.httpStatusCode === 404)`.
- `isPreconditionFailed`: `e instanceof S3ServiceException && (e.name === 'PreconditionFailed' || e.$metadata?.httpStatusCode === 412)`.
- Defensive: also accept plain `Error` with `$metadata` duck-typed (some S3-compatible backends throw non-SDK exceptions). Match on `name` first, fall back to status code.

### Step 4: Implement S3StorageAccessor

**File:** `src/s3-storage-accessor.ts`

```typescript
class S3StorageAccessor extends AbstractStorageAccessor {
  constructor(
    private readonly s3: S3Client,
    private readonly config: S3ProviderConfig,
  ) {}

  private objectKey(name: string): string {
    return this.config.objectPrefix + name;
  }

  async insertRecord(config: LockConfiguration): Promise<boolean> { /* ... */ }
  async updateRecord(config: LockConfiguration): Promise<boolean> { /* ... */ }
  async unlock(config: LockConfiguration): Promise<void> { /* ... */ }
  async extend(config: LockConfiguration): Promise<boolean> { /* ... */ }
}
```

Implement each method per spec §Locking Mechanism. Key points:

- **All four share a `HeadObject` helper** that returns `{ exists: boolean, etag?: string, metadata?: Record<string,string> }` — encapsulates the 404-vs-success split. Reduces duplication.
- **Metadata values are strings.** `Utils.toIsoString(...)` returns a string. GCS / S3 metadata is always string-typed. No conversion needed on write; on read, parse via `Date.parse(...)`.
- **`Body: ''`** — empty body. Set `ContentType: 'text/plain'` for tidiness; not required.
- **`IfNoneMatch: '*'`** for insert, **`IfMatch: etag`** for update/unlock/extend.
- Wrap `s3.send(...)` calls in try/catch; route to `isNotFound` / `isPreconditionFailed` helpers; re-throw everything else.

#### Helper: headObject

```typescript
private async headObject(name: string): Promise<{ etag?: string; metadata?: Record<string,string> } | null> {
  try {
    const out = await this.s3.send(new HeadObjectCommand({
      Bucket: this.config.bucket,
      Key: this.objectKey(name),
    }));
    return { etag: out.ETag, metadata: out.Metadata };
  } catch (e) {
    if (isNotFound(e)) return null;
    throw e;
  }
}
```

- `null` ⇒ object does not exist.
- Throws on any non-404 error.

#### Helper: parseLockUntil

```typescript
private parseLockUntil(metadata: Record<string,string> | undefined): number {
  const raw = metadata?.lockUntil;
  if (!raw) throw new LockException('Corrupted lock record: missing lockUntil');
  const ms = Date.parse(raw);
  if (Number.isNaN(ms)) throw new LockException(`Corrupted lock record: unparseable lockUntil '${raw}'`);
  return ms;
}
```

#### insertRecord

```typescript
async insertRecord(config: LockConfiguration): Promise<boolean> {
  const head = await this.headObject(config.name);
  if (head !== null) return false;
  try {
    await this.s3.send(new PutObjectCommand({
      Bucket: this.config.bucket,
      Key: this.objectKey(config.name),
      Body: '',
      Metadata: this.buildMetadata(config),
      IfNoneMatch: '*',
    }));
    return true;
  } catch (e) {
    if (isPreconditionFailed(e)) return false;
    throw e;
  }
}
```

#### updateRecord

```typescript
async updateRecord(config: LockConfiguration): Promise<boolean> {
  const head = await this.headObject(config.name);
  if (head === null) return false;
  const lockUntil = this.parseLockUntil(head.metadata);
  if (lockUntil > ClockProvider.now()) return false;
  try {
    await this.s3.send(new PutObjectCommand({
      Bucket: this.config.bucket,
      Key: this.objectKey(config.name),
      Body: '',
      Metadata: this.buildMetadata(config),
      IfMatch: head.etag,
    }));
    return true;
  } catch (e) {
    if (isPreconditionFailed(e)) return false;
    throw e;
  }
}
```

#### unlock

```typescript
async unlock(config: LockConfiguration): Promise<void> {
  const head = await this.headObject(config.name);
  if (head === null) return; // best-effort no-op
  try {
    await this.s3.send(new PutObjectCommand({
      Bucket: this.config.bucket,
      Key: this.objectKey(config.name),
      Body: '',
      Metadata: {
        lockUntil: Utils.toIsoString(unlockTime(config)),
        lockedAt: head.metadata?.lockedAt ?? Utils.toIsoString(config.createdAt),
        lockedBy: head.metadata?.lockedBy ?? this.getHostname(),
      },
      IfMatch: head.etag,
    }));
  } catch (e) {
    if (isPreconditionFailed(e)) return; // best-effort no-op, log warning
    throw e;
  }
}
```

#### extend

```typescript
async extend(config: LockConfiguration): Promise<boolean> {
  const head = await this.headObject(config.name);
  if (head === null) return false;
  const lockUntil = this.parseLockUntil(head.metadata);
  const lockedBy = head.metadata?.lockedBy;
  if (lockedBy !== this.getHostname()) return false;
  if (lockUntil <= ClockProvider.now()) return false;
  try {
    await this.s3.send(new PutObjectCommand({
      Bucket: this.config.bucket,
      Key: this.objectKey(config.name),
      Body: '',
      Metadata: {
        lockUntil: Utils.toIsoString(lockAtMostUntil(config)),
        lockedAt: head.metadata?.lockedAt ?? Utils.toIsoString(config.createdAt),
        lockedBy: head.metadata?.lockedBy ?? this.getHostname(),
      },
      IfMatch: head.etag,
    }));
    return true;
  } catch (e) {
    if (isPreconditionFailed(e)) return false;
    throw e;
  }
}
```

### Step 5: Implement S3LockProvider

**File:** `src/s3-lock-provider.ts`

```typescript
class S3LockProvider implements ExtensibleLockProvider {
  private readonly delegate: StorageBasedLockProvider;

  constructor(s3: S3Client, config: S3ProviderConfig) {
    const accessor = new S3StorageAccessor(s3, config);
    this.delegate = new StorageBasedLockProvider(accessor);
  }

  lock(config: LockConfiguration): Promise<SimpleLock | undefined> {
    return this.delegate.lock(config);
  }

  clearCache(name: string): void {
    this.delegate.clearCache(name);
  }
}
```

Thin wrapper — all behavior lives in `StorageBasedLockProvider` + `S3StorageAccessor`.

### Step 7: Unit tests (mocked S3Client)

**File:** `__tests__/unit/s3-storage-accessor.test.ts`

Mock `@aws-sdk/client-s3`'s `S3Client.send` to return canned `HeadObjectCommandOutput` / `PutObjectCommandOutput` / throw `S3ServiceException` with `$metadata.httpStatusCode`. Use `vi.mock` or instantiate a real `S3Client` and stub `send` via prototype.

Tests:
- `insertRecord`:
  - Happy path: HeadObject throws 404 NotFound → PutObject succeeds → returns `true`.
  - Object exists: HeadObject succeeds → returns `false` (no Put attempted).
  - Concurrent create: HeadObject 404 → PutObject throws 412 → returns `false`.
  - HeadObject throws 500 → propagates.
  - PutObject throws 500 → propagates.
- `updateRecord`:
  - Happy path: HeadObject returns ETag + lockUntil (expired) → PutObject IfMatch succeeds → `true`.
  - Still locked: HeadObject returns lockUntil in future → returns `false` (no Put).
  - Missing record: HeadObject 404 → returns `false`.
  - Concurrent modify: PutObject throws 412 → returns `false`.
  - Corrupt metadata: `lockUntil` missing → throws `LockException`.
- `unlock`:
  - Happy path: HeadObject returns ETag → PutObject IfMatch succeeds → resolves.
  - Missing record: HeadObject 404 → resolves (no-op).
  - Concurrent modify: PutObject 412 → resolves (no-op, logs warning).
- `extend`:
  - Happy path: HeadObject returns matching lockedBy + future lockUntil → PutObject succeeds → `true`.
  - Wrong owner: lockedBy mismatch → returns `false` (no Put).
  - Expired: lockUntil in past → returns `false` (no Put).
  - Missing record: 404 → returns `false`.
  - Concurrent modify: PutObject 412 → returns `false`.

**File:** `__tests__/unit/s3-lock-provider.test.ts`

- Construct `S3LockProvider` with a fake `S3Client` (or mock the accessor).
- Verify it delegates to `StorageBasedLockProvider`: first lock on a name triggers `insertRecord`, second lock on the same name triggers `updateRecord`.
- Verify `clearCache(name)` causes the next lock attempt to call `insertRecord` again.

**Caveats:**
- LocalStack ETag behavior: stable hash of content + metadata. Since body is empty, ETag changes only when metadata changes — exactly the semantics we want.
- If a LocalStack regression breaks `IfNoneMatch` / `IfMatch`, the integration test will fail loudly (412 fires at the wrong moment). This is the safety net.
- `forcePathStyle: true` is mandatory for LocalStack.

## Risks & Mitigations

| Risk | Mitigation |
|---|---|
| S3-compatible backends (MinIO, R2, older LocalStack) return different exception names / status codes for 412/404 | Error helpers match on both `name` and `$metadata.httpStatusCode`. Document tested backends in README. |
| `IfNoneMatch: "*"` not supported on older S3 or some compatible stores | Document minimum requirements. LocalStack ≥ 2.x and real AWS S3 support it. If user targets an unsupported store, `insertRecord` will always fail at the Put step — surface via integration test. |
| LocalStack regression on conditional puts breaks CI | Integration tests will fail loudly (good). Pin LocalStack image tag to a known-good version (`localstack/localstack:s3-3.0.2` or similar). |
| `HeadObject` Metadata is `undefined` if object has no metadata | Handled — `parseLockUntil` throws `LockException('Corrupted lock record')`. |
| Race between `HeadObject` and `PutObject` (TOCTOU) | Expected — the conditional put is the source of truth. `HeadObject` is only an optimization to avoid sending `PutObject` when we know it will fail. The conditional `IfMatch`/`IfNoneMatch` is the atomic check. |
| Lock objects accumulate forever | Documented non-goal. User configures S3 Lifecycle rule on `objectPrefix`. |
| ETag is an opaque string — cannot be parsed as a number | Treated as opaque string in `IfMatch`. No parsing. |
| AWS SDK v3 throws `NoSuchKey` instead of `NotFound` for HeadObject on some paths | Error helper accepts both names; matches `name === 'NotFound' || name === 'NoSuchKey'` and 404 status. |
| testcontainers pulls a large image | Pin to `localstack/localstack:s3-latest` (smaller than full image). Document `docker pull` in README. |
